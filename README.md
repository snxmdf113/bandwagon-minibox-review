## 三、MINIBOX-10-512 实测：三网回程路由与延迟到底怎么样

光看配置表没用，真正决定 MINIBOX 价值的是它那条 CN2 GIA 回程线路。我用了一年多，也跑了完整的 NextTrace 路由追踪，把数据摆出来给你看。

**电信回程**：从洛杉矶 DC99 出发，走 CN2 骨干网（AS4134 / 59.43.x.x 段），经过上海、北京节点落地，平均延迟在 130ms 上下。北京电信节点实测 148ms 左右，上海电信 128ms 左右。这条线路就是 MINIBOX 最大的底气——CN2 GIA 在高峰期也不容易拥堵，比普通 163 线路稳得多。

**联通回程**：同样从 CN2 骨干进入，再切换到联通 AS4837/AS4808 网络，落地北京联通 155ms 左右，上海联通 176ms 左右。比电信略慢一点，但相比普通 9929/4837 直连，走 CN2 中转的丢包和抖动都更小。

**移动回程**：从 CN2 骨干进入后切到 CMNET（AS9808），北京移动 155ms，上海移动 132ms，广东移动 161ms。移动的表现其实跟电信差不多，这条线路对三网都比较友好。

**CERNET 教育网**：从 CN2 转入 AS4538（CERNET），北京教育网 178ms 左右。校园网用户也能正常用，这点比一些只优化商业网的套餐要友好。

**CPU 与 IO**：bench.sh 测试显示 DC99 用的是 AMD EPYC 处理器，磁盘 IO 读写速度很快，远超普通 SATA SSD VPS。这台小鸡虽然内存小，但底子是 EPYC + RAID-10 SSD，硬件素质并不差。

**网络解锁**：实测 ChatGPT 是 DNS 解锁状态，流媒体解锁情况视具体 IP 而定。搬瓦工的 IP 池相对干净，原生 IP 的概率不算低。

一句话总结这台机器的表现：**配置不高但底子不差，线路给得大方但稳定性看脸**。它就是那种"够用、顺手、便宜"的小工具机，不是用来扛生产业务的。
